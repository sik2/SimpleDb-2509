### SimpleDB 만들기 과제 

simpleDb 객체는 멀티 쓰레드 환경에서 공유되어도 문제가 없어야 한다.

멀티 쓰레드 환경 예시 : 스프링 부트의 WebMVC 환경

DB Connection 객체는 simpleDb.close(); 를 하지 않는 이상 계속 열려있어야 한다.

여러개의 다른 쓰레드에서 동시에 simpleDb 객체를 사용해도 문제가 없어야 한다.

각 쓰레드는 독립적인 DB Connection 객체를 사용해야 한다.

즉 simpleDb 객체는 DB Connection 객체를 여러개 가지고 있어야 한다.

쓰레드 당 1개

힌트 : ThreadLocal or ThreadId + HashMap

### 코덱스가 해야할 일

1. 코덱스가 구동되면 AGENTS.md 문서와 REPORT.md 를 먼저 읽고 사용자 명령의 맥락을 파악한다
2. 사용자가 질문을 하면 설명을 최대한 알기쉽게 자세하게 해주고 코드는 사용자가 수정을 요청할때만 수정한다 
3. 사용자가 질문의 기록을 요청하면 질문에 대한 요약과, 자세한 설명을 정리해서 REPORT.md 에 번호를 매겨 마크다운 형식으로 순서대로 기재한다. 이 때 질문에 대한 설명을 핵심만 최대한 짧게 요약한다
4. 사용자가 질문을 삭제할수 있다는걸 가정하고 질문에 대한 번호를 인식해서 순서대로 기재한다.
5. 코드를 수정할 시에도 이와 마찬가지로 REPORT.md 에 수정된 코드와, 코드에 주석을 달아서 자세한 설명을 같이 마크다운 형식으로 기재한다.
